DeepSearch Document API 가이드

※본 가이드는 DeepSearch의 공식문서가 아니며, 정재광 과장이 DeepSearch API 문서를 참고하여 작성한 활용사례입니다.

이 가이드의 목적
한국거래소(KRX) 직원의 업무를 위해 작성되었습니다.

주요 관심사:
  • 뉴스 모니터링: 상장사 관련 뉴스 실시간 추적
  • 공시 의무 판단: 주요 경영사항 발생 여부 확인
  • 주가 변동 분석: 특정 뉴스와 주가 변동 상관관계 파악

2. 호출 파라미터 및 응답 필드

DocumentSearch 파라미터

DocumentSearch(
    category,              # 필수: 문서 카테고리 (news, research, company, patent)
    section,               # 필수: 세부 섹션 (economy, politics 등)
    query,                 # 필수: 검색 쿼리
    count=10,              # 한 페이지 문서 수 (기본 10)
    page=None,             # 페이지 번호
    date_from=None,        # 검색 시작일 (YYYYMMDD)
    date_to=None,          # 검색 종료일 (YYYYMMDD)
    summary=True,          # 문서 요약 여부
    clustering=False,      # 클러스터링 사용 여부
    uniquify=True,         # 중복 문서 제거
    highlight=False,       # 키워드 하이라이트
    fields=None            # 반환할 필드 지정
)

Proximity Search (근접 검색)

# 실무 예시: "회계" "부정"이 15단어 이내에 등장

개별 문서(doc) 필드

기본 정보
필드타입설명
uid / uid_strnumber/string문서 고유 ID
categorystring문서 카테고리 (news, company 등)
sectionstring세부 섹션 (economy, disclosure 등)
publisherstring출처 (언론사명, 증권사명 등)
authorstring작성자
titlestring제목
contentstring본문 요약 (원문 아님, DeepSearch가 생성한 요약본)
content_urlstring원문 링크
created_atstring문서 생성 일시
참고: content 필드는 DeepSearch가 생성한 요약본입니다. 실제 기사 원문이 아닙니다. 원문을 확인하려면 content_url 링크를 참조해야 합니다.
기업/엔티티 필드

securities (관련 종목)

상장기업만 매핑됨. 문서가 특정 상장기업에 대한 것으로 판단될 때 매핑
{
  "securities": [
    {
      "type": "company",
      "exchange": "KRX",
      "market": "KOSPI",           // KOSPI, KOSDAQ, KONEX
      "symbol": "005930",          // 종목코드
      "name": "삼성전자",           // 기업명
      "company_rid": "...",        // 법인등록번호
      "business_rid": "..."        // 사업자등록번호
    }
  ]
}

entities (관련 엔티티)

문서에서 추출된 주요 엔티티. 간략한 형태
{
  "entities": [
    {"type": "company", "name": "삼성전자"},
    {"type": "company", "name": "SK하이닉스"}
  ]
}

named_entities (언급 내용)

본문에서 언급된 모든 기업/인물/브랜드. 비상장기업 포함
{
  "named_entities": [
    {
      "type": "organization",      // company, person, brand 등
      "name": "카카오뱅크",
      "count": 5                   // 언급 횟수
    }
  ]
}
분석 정보

polarity (긍부정 분석)

securities가 매핑된 문서에 대해 긍부정 분석 수행
{
  "polarity": {
    "label": "1",        // 1: 긍정, 0: 중립, -1: 부정
    "name": "긍정",
    "score": 0.85        // 신뢰도 (0.0 ~ 1.0)
  }
}

esg (ESG 분석)

{
  "esg": {
    "category": {
      "label": "E",          // E: 환경, S: 사회, G: 지배구조, U: 해당없음
      "name": "환경",
      "score": 0.92
    },
    "polarity": {
      "label": "1",          // ESG 관점 긍부정
      "name": "긍정",
      "score": 0.78
    }
  }
}

industry (산업 분류)

{
  "industry": {
    "label": "C26",          // 한국표준산업분류 코드
    "name": "전자부품, 컴퓨터, 영상, 음향 및 통신장비 제조업",
    "score": 0.95
  }
}

3. 상장사 검색 필드 이해 - KRX 핵심

KRX 담당자가 반드시 알아야 할 핵심 개념
DeepSearch는 기사를 분석하여 세 가지 방식으로 기업 정보를 매핑합니다. 이 차이를 정확히 이해해야 원하는 검색 결과를 얻을 수 있습니다.

한눈에 보는 차이점

구분 securities entities named_entities
대상 상장기업만 주요 기업 모든 언급 (비상장 포함)
매핑 기준 문서가 해당 기업에 관한 것 문서에서 추출된 주요 기업 본문에 언급된 모든 것
정보량 상세 (종목코드, 시장, 등록번호) 간략 (이름, 타입) 중간 (이름, 타입, 언급횟수)
비상장 포함 ❌ ⚠️ 일부 ✅
인물 포함 ❌ ❌ ✅

활용사례 1: 상장사가 "주제"인 기사 찾기 (securities 사용)

securities 필드의 동작 원리

기사 본문
→
DeepSearch AI 분석
→
"이 기사의 주제가
특정 상장사인가?"
→
securities 매핑
핵심 포인트: "남성", "대상"은 일반적으로 흔히 쓰이는 단어입니다. 하지만 유가증권시장에 상장된 "(주)남성", "(주)대상"이라는 기업도 있습니다.

securities 필드로 검색하면, 단순히 "남성"이라는 단어가 언급된 기사가 아니라, DeepSearch가 "남성"이라는 상장사를 주제로 판단한 기사만 검색됩니다. 
# 상장사 "남성"이 주제인 기사 검색 (흔한 단어 "남성"이 포함된 기사는 제외됨)
DocumentSearch(["news"], ["economy"], "securities.name:남성")

# 상장사 "대상"이 주제인 기사 검색 (흔한 단어 "대상"이 포함된 기사는 제외됨)
DocumentSearch(["news"], ["economy"], "securities.name:대상")

# KOSPI/KOSDAQ 전체 상장사가 주제인 기사
DocumentSearch(["news"], ["economy"], "securities.market:(KOSPI or KOSDAQ)")

활용사례 2: 상장사가 "언급된" 기사 찾기 (named_entities 사용)

named_entities 필드의 동작 원리

기사 본문
→
NER(개체명 인식)
→
"이 기사에서
언급된 기업은?"
→
named_entities 매핑
핵심 포인트: 기사의 주제가 아니더라도, 본문에서 언급만 되어도 검색됩니다. 비상장 기업, 인물, 브랜드도 포함됩니다. 
# 삼성전자가 "언급된" 모든 기사 (주제가 아니어도 됨)
DocumentSearch(["news"], ["economy"], "named_entities.entities.company.name:삼성전자")

# 특정 인물이 언급된 기사
DocumentSearch(["news"], ["economy"], "named_entities.entities.person.name:이재용")

# 비상장 기업이 언급된 기사 (쿠팡은 미국 상장)
DocumentSearch(["news"], ["economy"], "named_entities.entities.company.name:쿠팡")

# 삼성전자의 비상장 종속회사가 언급된 기사 검색

# 여러 종속회사 동시 검색
DocumentSearch(

4. KRX 실무활용예시

이 섹션에서는 DeepSearch API를 활용한 실제 KRX 업무 시나리오를 다룹니다.
각 시나리오는 다이어그램 → 설명 → 코드 순서로 직관적으로 이해할 수 있도록 구성되어 있습니다.
이 섹션의 내용
  • 1. 상장사 주제 + 키워드
  • 2. 긍부정 필터링
  • 3. 일일 모니터링
  • 4. 공시 의무 탐지
  • 5. 테마/이슈 모니터링
  • 6. 실시간 이슈
  • 7. 비상장사 재무수치
  • 유틸리티 함수

시나리오 1: 특정 상장사 주제 기사 + 키워드 검색

목적: DeepSearch가 특정 상장사 "주제"로 판단한 기사 중, 특정 키워드가 포함된 기사만 찾기

검색 흐름

전체 뉴스
→
securities로
상장사 주제 필터
→
title/content에서
키워드 검색
→
최종 결과
중요한 차이점:
• securities: DeepSearch가 사전에 AI로 분석하여 해당 상장사 주제 기사로 판단한 것
• title/content 키워드: 단순 Ctrl+F에 해당하는 문자열 검색

참고: content 필드는 기사 원문이 아닌 DeepSearch가 생성한 요약본입니다. 원문 전체를 대상으로 키워드 검색하는 것이 아님에 유의하세요.
코드 보기 
# 삼성전자 주제 기사 중 "수주"가 언급된 기사 (제목+요약본 검색)
DocumentSearch(
    ["news"], ["economy"],
    "securities.name:삼성전자 and (title:수주 or content:수주)"
)

# KOSPI 상장사 주제 기사 중 "실적" 키워드가 포함된 기사
DocumentSearch(
    ["news"], ["economy"],
    "securities.market:KOSPI and (title:실적 or content:실적)"
)

# 제목에서만 키워드 검색 (더 정확한 필터링)
DocumentSearch(
    ["news"], ["economy"],
    "securities.name:현대자동차 and title:리콜"
)

def filter_negative_news(docs):
    """부정 뉴스 필터링 후처리"""
    return [
        {

def theme_monitoring(theme_keywords, markets=['KOSPI', 'KOSDAQ']):
    """특정 테마 관련 상장사 뉴스 모니터링

    Args:
        theme_keywords: ['AI', '인공지능', '생성AI', 'ChatGPT']
        markets: ['KOSPI', 'KOSDAQ']
    """
    from datetime import datetime, timedelta

    # 1. 테마 키워드 쿼리 구성 (title/content 검색)
    theme_query = " or ".join([f'(title:"{kw}" or content:"{kw}")' for kw in theme_keywords])
    market_query = " or ".join(markets)

    query = f"securities.market:({market_query}) and ({theme_query})"

    # 2. 최근 1주일 뉴스 검색
    docs = DocumentSearch(
        ["news"], [],
        query,
        count=100,
        date_from=(datetime.now() - timedelta(days=7)).strftime("%Y%m%d")
    )

    # 3. 기업별 언급 빈도 집계
    company_mentions = {}
    for doc in docs['docs']:
        for sec in doc.get('securities', []):
            name = sec['name']
            symbol = sec['symbol']
            key = f"{name}({symbol})"

            if key not in company_mentions:
                company_mentions[key] = {
                    'count': 0,
                    'positive': 0,
                    'negative': 0
                }

            company_mentions[key]['count'] += 1

            pol = doc.get('polarity', {}).get('name', '중립')
            if pol == '긍정':
                company_mentions[key]['positive'] += 1
            elif pol == '부정':
                company_mentions[key]['negative'] += 1

    # 4. 언급 빈도순 정렬
    sorted_companies = sorted(
        company_mentions.items(),
        key=lambda x: x[1]['count'],
        reverse=True
    )

    return sorted_companies[:20]

def realtime_issue_monitoring():
    """실시간 트렌딩 이슈 중 상장사 관련 건 모니터링"""

    # 1. 현재 트렌딩 토픽 조회
    trending = SearchTrendingTopics(["news"], ["economy"])

    # 2. 상장사 관련 토픽 필터링
    stock_related_topics = []

    for topic in trending['topics']['news']['economy']:
        securities = topic.get('statistics', {}).get('securities', [])

        if securities:
            stock_related_topics.append({
                'rank': topic['rank'],
                'topic': topic['topic'],
                'score': topic['score'],
                'keywords': [k['keyword'] for k in topic.get('keywords', [])[:5]],
                'companies': [f"{s['name']}({s['market']})" for s in securities[:5]],
                'sentiment': topic.get('statistics', {}).get('polarities', [])
            })

    # 3. 부정적 이슈 우선 정렬
    def has_negative_sentiment(topic):
        for pol in topic.get('sentiment', []):
            if pol.get('label') == '-1' and pol.get('doc_count', 0) > 0:
                return True
        return False

    negative_first = sorted(
        stock_related_topics,
        key=lambda x: (has_negative_sentiment(x), -x['score']),
        reverse=True
    )

    return negative_first

def analyze_financial_trends(entity_code, years=5):
    """비상장사 주요 재무지표 추이 분석"""
    import pandas as pd
    from datetime import datetime, timedelta

    end_date = datetime.now().strftime("%Y-%m-%d")
    start_date = (datetime.now() - timedelta(days=365*years)).strftime("%Y-%m-%d")

    # 재무제표 조회
    fs = GetFinancialStatements(
        entity_code,
        report_type="IFRS",
        consolidated=True,
        is_annual=True,
        date_from=start_date,
        date_to=end_date
    )

    # 주요 계정 코드 (DeepSearch 재무계정 기준)
    key_accounts = {
        '1100': '현금및현금성자산',
        '1300': '매출채권',
        '2900': '자산총계',
        '4000': '부채총계',
        '5000': '자본총계',
        '6000': '매출액',
        '6500': '영업이익',
        '7000': '당기순이익'
    }

    # 연도별 주요 지표 추출
    trends = {}
    for account_id, account_name in key_accounts.items():
        account_data = [
            row for row in fs
            if row['account_id'] == account_id
        ]
        trends[account_name] = {
            row['date'][:4]: row['value']
            for row in account_data
        }

    return trends

def check_listing_requirements(entity_code):
    """상장예비심사 재무요건 체크"""

    fs = GetFinancialStatements(
        entity_code,
        report_type="IFRS",
        consolidated=True,
        is_annual=True,
        date_from="2021-01-01"
    )

    # 최근 3개년 추출
    recent_years = {}
    for row in fs:
        year = row['date'][:4]
        if year not in recent_years:
            recent_years[year] = {}
        recent_years[year][row['account_id']] = row['value']

    # 코스피 상장요건 체크 예시
    checks = {
        '자기자본': [],
        '매출액': [],
        '영업이익': [],
        '당기순이익': []
    }

    for year in sorted(recent_years.keys())[-3:]:
        data = recent_years[year]
        checks['자기자본'].append(data.get('5000', 0))  # 자본총계
        checks['매출액'].append(data.get('6000', 0))
        checks['영업이익'].append(data.get('6500', 0))
        checks['당기순이익'].append(data.get('7000', 0))

    return {
        '자기자본_최근': checks['자기자본'][-1] if checks['자기자본'] else 0,
        '매출액_3년평균': sum(checks['매출액']) / len(checks['매출액']) if checks['매출액'] else 0,
        '영업이익_흑자연속': all(v > 0 for v in checks['영업이익']),
        '당기순이익_흑자연속': all(v > 0 for v in checks['당기순이익'])
    }

def analyze_sentiment_distribution(docs):

def analyze_company_mentions(docs):
    """기업별 언급 빈도 분석"""
    mentions = Counter()
    for doc in docs:
        for ne in doc.get('named_entities', []):
            if ne.get('type') == 'company':
                mentions[ne['name']] += ne.get('count', 1)
    return mentions.most_common(20)
